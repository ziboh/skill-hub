# Skill 启用/停用开关设计

日期：2026-07-14

## 目标

为项目 Skill 和 Agent Skill 提供独立启用/停用开关。停用时不删除 Skill 内容，同时让依赖标准 `SKILL.md` 文件名的其他扫描器无法识别该 Skill。

## 交互

- 每个 Skill 卡片左上角显示独立开关。
- 保持当前卡片主体布局不变，利用左上角现有留白放置开关，不新增顶部行。
- 开启状态继续使用标准 `SKILL.md` 文件名。
- 关闭状态在原 Skill 目录内将 `SKILL.md` 改名为 `SKILL.md.disabled`。
- 停用项仍在项目 Skill / Agent Skill 页面中显示，开关可再次打开。
- 停用项不进入全局 Skill inventory、我的 Skill 或其他只使用默认扫描结果的识别路径。

## 技术边界

- 默认 `scanForSkillFiles()` 只扫描精确文件名 `SKILL.md`，因此停用文件不会被普通 Skill 扫描识别。
- 管理页面使用包含停用项的扫描模式，返回 `enabled` 状态供 UI 展示。
- 对目录软链接优先处理链接入口，避免改写被链接的源目录；普通目录则改名 Skill 文件。
- 启用/停用操作必须是原子化的单文件重命名，检测目标冲突后才执行。
- 不使用浏览器内置弹窗，异常通过现有 Toast 提示。

## 验收标准

1. 项目 Skill 和 Agent Skill 卡片左上角均能独立切换启用状态。
2. 停用后 Skill 目录和内容仍存在。
3. 停用后目录内没有标准 `SKILL.md`，默认扫描结果不包含该 Skill。
4. 页面刷新、应用重启后停用状态保持不变。
5. 重新启用后恢复标准 `SKILL.md`，默认扫描重新识别。
6. 现有卡片标签、标题、描述和操作按钮布局不发生结构性变化。
